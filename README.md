### 批量视频裁剪片尾工具（H.264 转码版）使用说明

#### 工具简介
这是一款基于 Python 和 FFmpeg 开发的智能视频批量处理工具。它能够自动扫描视频末尾指定时长，通过图像识别技术定位片尾（结束标志），并自动执行裁剪操作。

本版本采用“先匹配后转码”的策略，将所有输出视频统一转换为兼容性最好的 H.264 编码格式，有效解决了老旧编码格式（如 MPEG-4）的兼容性问题，同时保证了裁剪的时间点精准度。

#### 核心功能
- **智能图像匹配**：基于 OpenCV 的模板匹配算法，自动识别视频结尾的特定画面。
- **统一转码输出**：强制将输出视频编码为 H.264 (MP4)，确保在任何设备或平台上均可流畅播放。
- **GPU加速**：对于 NVIDIA 显卡（目前最主流），我们将编码器从 libx264（CPU 编码）改为 h264_nvenc（NVIDIA 硬件编码）。这通常能带来 5-10 倍 的速度提升。如果显卡不支持或驱动有问题，可以选择 CPU 模式，防止程序崩溃。
- **批量自动化**：支持文件夹级批量处理，自动遍历输入目录下的所有常见视频格式。
- **可视化进度**：运行过程中实时显示匹配进度、相似度数值及处理结果。
- **深度匿名处理**：
    - 使用 -map_metadata -1 丢弃所有元信息。
    - 使用 -movflags +faststart 重写文件原子结构，清除私有数据。
- **隐私保护**：生成的视频无法通过文件属性或哈希值追溯到原始来源。

#### 环境准备
在运行本工具之前，请确保您的计算机已配置好以下运行环境：

- **Python 环境**：需安装 Python 3.6 或更高版本。
- **FFmpeg 组件**：
    - 必须安装 FFmpeg 并将其添加到系统的 PATH 环境变量中。
    - 验证方法：在命令行输入 `ffmpeg -version`，若能显示版本信息即代表配置成功。
- **依赖库安装**：
    - 打开终端或命令行窗口，执行以下命令安装必要的 Python 库：
    - `pip install opencv-python numpy`

#### 目录结构
建议按照以下结构组织您的项目文件，以确保路径配置正确：

```text
project_root/
├── batch_video_cutter.py    # 主程序脚本
├── end.png                  # [关键] 结束标志图片（需自行准备）
├── input_videos/            # 待处理的视频文件夹
└── output_videos/           # 裁剪完成后的输出文件夹
```

#### 配置与运行
**第一步：准备结束标志图片**

截取您视频中出现的静态片尾画面（例如公司 Logo、结束卡片等），保存为 `end.png`，并将其放置在脚本同级目录下。图片清晰度应与视频画面保持一致。

可以使用剪映导出静帧画面的方式获取。
如果片尾非图片，而是视频，那么需要片尾视频的第一帧画面。

**第二步：修改脚本配置**
使用文本编辑器打开 `batch_video_cutter.py`，找到文件底部的“配置区域”，根据您的实际需求修改以下参数：

- **INPUT_DIR**：填写包含待处理视频的文件夹绝对路径（例如 `D:\Videos\Raw`）。
- **OUTPUT_DIR**：填写处理后视频的保存路径（例如 `D:\Videos\Done`）。
- **END_IMAGE**：结束标志图片的文件名（默认为 `end.png`）。
- **SEARCH_TIME**：搜索范围（秒）。指在视频的最后多少秒内寻找标志。建议设置为片尾时长的 1.5 倍左右（例如片尾约 5 秒，此处可设为 8-10 秒）。

**第三步：执行脚本**
在命令行中运行脚本：
`python batch_video_cutter.py`

程序启动后会自动检查 FFmpeg 状态，随后开始批量处理。处理过程中，控制台会输出详细的日志信息。

#### 工作流程解析
本工具的处理逻辑分为三个核心阶段：

**阶段一：画面扫描与定位**
程序首先使用 OpenCV 读取原视频，根据设定的搜索时长（如最后 10 秒），从后向前或从指定位置开始逐帧扫描。通过计算视频帧与 `end.png` 的相似度（阈值设为 0.85），精准定位结束标志首次出现的时间点。

**阶段二：参数计算**
一旦匹配成功，程序会立即锁定该帧的时间戳（例如第 125.5 秒），并将其作为裁剪的终点。若未找到匹配画面，程序会跳过该文件并记录日志。

**阶段三：转码与裁剪**
调用 FFmpeg 执行核心操作。程序不会简单地复制流，而是将视频重新编码为 H.264 格式，并在编码过程中截断至锁定的时间点。这不仅去除了片尾，还修复了原视频可能存在的编码不规范问题。

#### 常见问题排查
- **报错“未找到 FFmpeg”**：
    - 请检查 FFmpeg 是否已正确安装，并确认 `bin` 目录已添加到系统环境变量 Path 中。重启命令行窗口后再试。
- **匹配失败或找不到标志**：
    - 检查 `end.png` 是否与视频结尾的画面完全一致（包括分辨率比例）。
    - 尝试调整代码中的 `threshold` 变量（默认 0.85），降低数值（如 0.75）可提高识别宽容度。
    - 确认 `SEARCH_TIME` 设置的时间长度是否覆盖了片尾出现的时间段。
- **输出视频画面花屏或不同步**：
    - 本工具已通过添加 `-avoid_negative_ts 1` 参数规避了大部分时间戳问题。若仍出现此情况，可能是原视频本身存在严重的帧率异常，建议检查原视频文件。
