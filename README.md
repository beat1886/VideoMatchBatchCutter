### 批量视频裁剪片尾工具（多进程 H.264 转码版）使用说明

#### 工具简介

这是一款基于 Python、OpenCV 和 FFmpeg 开发的高性能智能视频批量处理工具。它能够自动扫描视频末尾指定时长，通过图像识别技术定位片尾（结束标志），并自动执行裁剪操作。

本版本引入了**多进程并行处理**与**交互式菜单**，支持 CPU 多核并行、GPU 硬件加速以及单核稳定模式。采用“先匹配后转码”策略，将所有输出视频统一转换为兼容性最好的 H.264 编码格式，在确保裁剪时间点精准的同时，极大提升了大批量视频的处理效率。

#### 核心功能

- **智能图像匹配**：基于 OpenCV 的模板匹配算法，自动识别视频结尾的特定画面（阈值 0.85）。

- 多模式运行

  ：

  - **CPU 多核并行**：自动调用电脑所有核心同时处理，速度最快。
  - **GPU 硬件加速**：支持 NVIDIA 显卡（NVENC），极大降低 CPU 占用。
  - **CPU 单核非并行**：最稳定的传统模式，日志输出清晰有序。

- **统一转码输出**：强制将输出视频编码为 H.264 (MP4)，确保在任何设备或平台上均可流畅播放。

- **可视化交互**：内置实时进度条（tqdm），动态显示处理进度、当前任务及最终统计报告。

- **深度隐私保护**：使用 `-map_metadata -1` 丢弃所有元信息，生成的视频无法通过文件属性追溯到原始来源。

#### 环境准备

在运行本工具之前，请确保您的计算机已配置好以下运行环境：

1. **Python 环境**：需安装 Python 3.6 或更高版本。

2. **FFmpeg 组件**：必须安装 FFmpeg 并将其添加到系统的 PATH 环境变量中。（验证方法：命令行输入 `ffmpeg -version`）

3. 依赖库安装

   ：打开终端或命令行窗口，执行以下命令安装必要的 Python 库：

   ```bash
   pip install opencv-python numpy tqdm
   ```

   (注：tqdm 用于显示漂亮的进度条)

#### 目录结构

建议按照以下结构组织您的项目文件：

```text
project_root/
├── batch_video_cutter.py        # 主程序脚本
├── end.png                      # [关键] 结束标志图片（需自行准备）
├── input_videos/                # 待处理的视频文件夹
├── output_videos/               # 裁剪完成后的输出文件夹（自动生成）
└── 一键清空输入和输出目录.bat    # [工具] 双击即可秒删清空上述两个文件夹
```

#### 配置与运行

**第一步：准备结束标志图片**
截取视频中出现的静态片尾画面（如 Logo、结束卡片），保存为 `end.png` 并放置在脚本同级目录下。图片清晰度应与视频画面保持一致。

**第二步：修改脚本配置**
使用文本编辑器打开脚本，找到文件顶部的“配置区域”，根据实际需求修改以下参数：

- `INPUT_DIR`：待处理视频的文件夹路径（默认为脚本目录下的 `input_videos`）。
- `OUTPUT_DIR`：处理后视频的保存路径（默认为脚本目录下的 `output_videos`）。
- `END_IMAGE`：结束标志图片的文件名（默认为 `end.png`）。
- `SEARCH_TIME`：搜索范围（秒）。指在视频的最后多少秒内寻找标志。

**第三步：执行脚本**
在命令行中运行脚本：

```bash
python batch_video_cutter.py
```

程序启动后会出现交互式菜单，您可以根据需求选择运行模式：

- **选项 1**：CPU 多核并行（极速，适合大批量处理）。
- **选项 2**：GPU 硬件加速（需 NVIDIA 显卡）。
- **选项 3**：CPU 单核非并行（最稳，日志输出最清晰）。

#### 配套清理工具

为了方便日常测试与迭代，项目附带了**“一键清空输入和输出目录.bat”**清理工具。

- **使用方法**：直接双击该 `.bat` 文件。
- **功能说明**：脚本会瞬间静默清空 `input_videos` 和 `output_videos` 文件夹内的所有文件与子目录（不会删除文件夹本身），无需任何确认，双击即完成清理。

#### 工作流程解析

1. **画面扫描与定位**：程序使用 OpenCV 读取原视频，在设定的搜索时长内逐帧扫描，计算与 `end.png` 的相似度。
2. **参数计算**：一旦匹配成功（相似度≥0.85），立即锁定该帧的时间戳作为裁剪终点。
3. **多进程分发**：根据选择的模式，将任务分发给单个或多个进程同时执行。
4. **转码与裁剪**：调用 FFmpeg 将视频重新编码为 H.264 格式，并截断至锁定的时间点。

#### 常见问题排查

- **报错“未找到 FFmpeg”**：请检查 FFmpeg 是否已正确添加到系统环境变量 Path 中，并重启命令行窗口。

- 匹配失败或找不到标志

  ：

  - 检查 `end.png` 是否与视频结尾画面完全一致。
  - 确认 `SEARCH_TIME` 设置的时间长度是否覆盖了片尾出现的时间段。

- **进度条显示异常**：请确保已安装 `tqdm` 库 (`pip install tqdm`)。

- **终端打印乱码**：多进程模式下打印偶尔会交错，属正常现象，最终报告会按文件顺序整齐输出。
